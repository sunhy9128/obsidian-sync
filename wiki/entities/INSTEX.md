# INSTEX（贸易互换支持工具）

> Instrument in Support of Trade Exchanges — 欧盟绕开美元制裁伊朗的"非美元清算"实验

**INSTEX**（Instrument in Support of Trade Exchanges，贸易互换支持工具）是法国、德国、英国（2019 年退出后）于 2019 年 1 月联合发起的特殊目的载体（SPV），总部设在巴黎。其核心目标是建立绕开美国金融制裁的"非美元清算通道"，使欧盟企业能够继续与伊朗进行人道主义贸易（药品、医疗器械、食品等），同时规避美国对与伊朗交易的第三国企业的次级制裁。

INSTEX 是欧盟在"美欧战略分歧"背景下的关键金融基础设施试验，其成败反映了欧盟追求"战略自主"的实际能力。

## 二、成立背景

### 2.1 美欧在伊朗问题上的分歧

```
【2015 年伊核协议（JCPOA）】
├── 伊朗：承诺限制核计划
├── 美/英/法/德/俄/中：解除对伊制裁
└── 特朗普（2017 上任）：单方面退出

【2018 年特朗普重启对伊制裁】
├── 美国：全面切断伊朗 SWIFT + 全面经济制裁
├── 欧盟：坚持伊核协议有效，希望继续与伊朗贸易
└── 美欧战略分歧公开化

【欧盟的两难】
├── 1. 安全依赖美国（北约 + 情报共享）
├── 2. 经济上又需维护自身利益
├── 3. 多数欧盟企业因长臂管辖被迫退出伊朗
└── 4. 缺乏独立金融基础设施
```

### 2.2 INSTEX 的设计目标

```
【三重目标】
├── 1. 维持欧盟与伊朗的人道主义贸易
├── 2. 规避美国长臂管辖（次级制裁）
└── 3. 推动欧盟"战略自主"与"非美元清算"探索

【设计原则】
├── 不使用美元
├── 不通过 SWIFT（理论上）
├── 进出口对等匹配（双边易货机制）
└── 仅限人道主义商品（药品、医疗、食品、农业）
```

## 三、运作机制

### 3.1 核心创新：易货清算

```
【传统跨境支付模式】
欧盟进口商 → 银行 → SWIFT → 美元清算 → 伊朗出口商
              ↓
         美元通道（被美国切断）

【INSTEX 模式】
欧盟进口商 → INSTEX → 抵消余额 → INSTEX 伊朗方 → 伊朗出口商
              ↓                       ↓
        欧元记账（不进 SWIFT）    伊朗里亚尔记账
              ↓                       ↓
        欧盟商品出口           欧盟商品进口
              ↓                       ↓
        抵消双边贸易差额       实际资金不跨境
```

### 3.2 详细的"易货"流程

```
【典型交易示例：欧盟药品 ↔ 伊朗石油/石化产品】

步骤 1：欧盟出口商向伊朗出售药品（人道主义）
        → 欧盟出口商在 INSTEX 登记"应收"

步骤 2：伊朗出口商向欧盟出售石化产品（非制裁）
        → 伊朗出口商在伊朗版 INSTEX 登记"应收"

步骤 3：INSTEX 内部抵消（双边净额清算）
        → 实际资金不跨境流动

步骤 4：完成"虚拟清算"
        → 欧盟药品到达伊朗，伊朗石化产品到达欧盟
```

### 3.3 关键设计特征

| 特征 | 说明 | 目的 |
|------|------|------|
| **不碰美元** | 完全使用欧元 + 伊朗里亚尔 | 规避美国长臂管辖 |
| **不通过 SWIFT** | 内部记账式清算 | 减少被美国追踪的风险 |
| **进出口对等** | 净额清算，双边平衡 | 减少实际资金流动 |
| **人道主义限定** | 仅限药品/食品/医疗 | 降低政治敏感性 |
| **私营化运营** | 由 SPV 而非政府机构运营 | 规避主权风险 |

## 四、发展历程

### 4.1 关键里程碑

| 时间 | 事件 |
|------|------|
| 2018-05 | 特朗普宣布退出伊核协议 |
| 2018-08 | 美方首批次级制裁生效 |
| 2019-01 | 法德英宣布成立 INSTEX |
| 2019-03 | INSTEX 正式注册成立 |
| 2019-11 | 英国退出（受美国压力）|
| 2020-03 | 完成首笔交易（医疗物资）|
| 2020-08 | 比利时、丹麦、芬兰、荷兰、挪威、瑞典加入 |
| 2021-至今 | 实际交易极为有限，机构运作停滞 |

### 4.2 英国退出的意义

```
【2019 年 11 月英国退出 INSTEX】

表面原因：英国大选 / 政治更迭
深层原因：
├── 1. 美英特殊关系压力（"五眼联盟"依赖）
├── 2. 伦敦金融城高度依赖美元清算
├── 3. 英国企业长臂管辖风险敏感度高于法德
└── 4. INSTEX 实际效能远低于预期

战略意义：
├── 英国退出 = 欧盟"战略自主"的一次挫败
├── 暴露欧盟内部对美国压力的不对称承受力
└── 强化了"非美元清算"建设的紧迫性
```

### 4.3 首笔交易的"象征意义大于实质"

```
【2020 年 3 月首笔交易】
├── 金额：极小（具体未公开）
├── 货物：德国医疗物资
├── 实际效果：仅具政治象征意义
├── 后续交易：基本停滞
└── 机构现状：半瘫痪，象征性存在
```

## 五、INSTEX 的结构性失败

### 5.1 失败的五大原因

```
【失败原因 ①：人道主义商品的"双向贸易"难匹配】
├── 伊朗需要进口：药品、医疗器械
├── 但伊朗能向欧盟出口的：石油（被制裁）、石化
├── 进出口对等 → 难以找到匹配的"易货"组合
└── 实际可交易品类极为有限

【失败原因 ②：欧盟企业的合规恐惧】
├── 美国长臂管辖 = 美元清算账户冻结
├── 任何欧盟企业参与 INSTEX 都面临被美国"次级制裁"风险
├── 风险收益比极差 → 企业不愿参与
└── 多数欧盟企业早在 2018 年就退出伊朗

【失败原因 ③：欧盟内部对美国的依赖差异】
├── 安全依赖美国：北约、情报、军售
├── 经济依赖美国：金融市场、技术
├── 政治依赖美国：俄乌、中东议题需要美方支持
└── 法德愿意"硬刚"，但实际承受力有限

【失败原因 ④：基础设施缺陷】
├── 缺乏与伊朗的有效银行通道
├── 净额清算机制设计复杂，实际操作困难
├── 没有对应的伊朗版 INSTEX 实体支持
└── 与 CIPS/SPFS 等成熟系统差距巨大

【失败原因 ⑤：政治意愿不足】
├── 欧盟内部对 INSTEX 的政治支持有限
├── 各国央行、商业银行被动配合
├── 缺乏类似中俄"金融核威慑"的战略决心
└── INSTEX 沦为"政治表态工具"
```

### 5.2 INSTEX vs CIPS/SPFS 的关键差异

| 维度 | INSTEX | CIPS | SPFS |
|------|--------|------|------|
| 主导方 | 欧盟（多国协调）| 中国（央行主导）| 俄罗斯（央行主导）|
| 政治决心 | 弱（易受美方压力）| 强（国家战略）| 极强（生死存亡）|
| 基础设施 | 极简（SPV 形式）| 完善（央行清算系统）| 完整（央行清算系统）|
| 实际效能 | 几乎为零 | 日均 $800 亿 | 内部清算 + 周边协同 |
| 抗压能力 | 低 | 中 | 高 |

## 六、INSTEX 的历史意义

### 6.1 作为"思想实验"的价值

> [!key-insight] INSTEX 是欧盟"战略自主"的第一次金融尝试
>
> 尽管 INSTEX 几乎完全失败，但它是**欧盟首次尝试建立独立于美国金融体系的清算基础设施**。其失败本身揭示了"非美元清算"建设的三个条件缺一不可：
> 1. **政治决心**（愿意承受美国制裁）
> 2. **实体基础设施**（完整的清算系统）
> 3. **市场基础**（足够的双边贸易量）
>
> INSTEX 三者都不足，所以失败。

### 6.2 催生了哪些后续机制

```
【INSTEX 的"遗产"】
├── 1. 欧盟"反长臂管辖"立法（Blocking Statute）
├── 2. 数字欧元研发加速
├── 3. 欧盟"自主清算"概念被反复讨论
├── 4. 中欧在"非美元清算"上的间接呼应
└── 5. 后续可能推出 INSTEX 2.0（数字版）
```

### 6.3 与 CIPS 的对比启示

```
【为什么 CIPS 成功而 INSTEX 失败？】

CIPS 的优势：
├── 1. 中国央行直接主导 → 政治决心强
├── 2. 拥有 14 亿人口市场 → 内循环足够
├── 3. 一带一路提供外部需求 → 不依赖欧洲
├── 4. 国家资本输出 → 项目融资支撑
└── 5. 人民币国际化作为长期战略 → 持续投入

INSTEX 的劣势：
├── 1. 欧盟多国协调 → 政治决心被稀释
├── 2. 欧盟市场对外依赖度高 → 内循环不足
├── 3. 缺乏类似一带一路的外部支撑 → 需求不足
├── 4. 私营企业主导 → 没有国家资本背书
└── 5. 缺乏长期战略承诺 → 沦为短期工具
```

## 七、未来展望

### 7.1 INSTEX 2.0 的可能形态

```
【INSTEX 的潜在演进】
├── 1. 数字化升级（基于 CBDC 的清算）
├── 2. 扩展合作范围（不仅限伊朗）
├── 3. 与数字欧元整合
├── 4. 与 CIPS / mBridge 对接
└── 5. 转向"绿色贸易"等更安全主题
```

### 7.2 欧盟在"非美元清算"中的真实地位

```
【欧盟的尴尬处境】
├── 经济上：希望"战略自主"
├── 安全上：仍依赖美国（北约、情报、军售）
├── 金融上：缺乏独立基础设施
├── 政治上：内部对美国的依赖不对称
└── 结论：欧盟在"非美元清算"中只能发挥"协调方"而非"主导方"的作用
```

### 7.3 INSTEX 的最终命运

```
【可能的三种结局】
├── 结局 ①：长期象征性存在
├── 结局 ②：升级为数字版"INSTEX 2.0"
├── 结局 ③：被新的"反制裁"机制取代
└── 最可能：长期处于"半死不活"状态，但具有政治象征意义
```

## 八、关键洞察

> [!key-insight] INSTEX 是"政治意志测试"
>
> INSTEX 的失败揭示了一个深刻的现实：**非美元清算基础设施的建设，需要的不只是金融技术，更是政治意志**。欧盟缺乏这种意志，所以失败；中俄有，所以成功。

> [!key-insight] INSTEX 的"边际效用"
>
> 即便 INSTEX 几乎完全失败，它的存在也具有边际效用：
> - 表明欧盟愿意尝试"非美元清算"
> - 为后续欧盟金融基础设施提供经验教训
> - 增加美国制裁成本（每次制裁需要考虑 INSTEX 等规避渠道）
> - 是"金融威慑平衡"的微小一极

> [!contradiction] "人道主义贸易"的悖论
>
> INSTEX 限定人道主义贸易，本意是降低政治敏感性，但实际上：
> - 减少了可交易品类
> - 增加了"双向匹配"的难度
> - 让企业参与的动机更弱
> - 最终沦为"政治正确但无实际效用"的工具
