# 操作风险

> 因内部流程、系统故障或人为失误导致的损失风险

## 核心定义

操作风险（Operational Risk）是指由于不完善或有问题的内部程序、人员、系统或外部事件而造成损失的风险。与信用风险和市场风险不同，操作风险更难以量化，但其损失往往是**厚尾分布**——低频高损的极端事件构成最大威胁。

## 主要类型

### 七类事件（Basel 分类）

| 类别 | 举例 |
|------|------|
| **内部欺诈** | 交易员虚假交易、员工贪污 |
| **外部欺诈** | 黑客盗取客户资产、网络诈骗 |
| **就业政策和工作场所安全** | 歧视、骚扰、职场安全事故 |
| **客户、产品和业务做法** | 不当销售、违反信义义务 |
| **实物资产损坏** | 地震、火灾、恐怖袭击 |
| **业务中断和系统故障** | 系统宕机、网络攻击 |
| **执行、交割和流程管理** | 数据录入错误、交割失败 |

### 损失分布特征

操作风险损失呈**肥尾（Fat Tail）**特征：

- 99% 的事件是小额损失（如系统故障、数据错误）
- 1% 的事件是大额灾难性损失（如交易欺诈、模型错误）
- 正态假设严重低估尾部风险

**典型案例：**
- 摩根大通 "London Whale"（2012）：60亿美元交易损失
- 瑞银交易欺诈（2011）：23亿美元
- 巴林银行（1995）：14亿美元倒闭

## 计量方法

### 四种方法对比

| 方法 | 全称 | 复杂度 | 适用范围 |
|------|------|--------|----------|
| **BIA** | 基本指标法 | 最低 | 小型银行 |
| **SA** | 标准法 | 低 | 中型银行 |
| **ASA** | 替代标准法 | 中 | 中型银行 |
| **AMA** | 高级计量法 | 高 | 大型银行 |

### 基本指标法（BIA）

```
操作风险资本 = α × Gross Income（总收入）
α = 15%（监管固定系数）
总收入 = 净利息收入 + 非利息收入（剔除异常项目）
```

特点：简单但过于粗糙，未反映银行实际风险特征。

### 标准法（SA）

将业务线分为 8 个，分别计算：

```
操作风险资本 = Σ max[Σ(GI_1×β_1) + ... + Σ(GI_8×β_8), 0]
```

各业务线 β 系数：
- 公司金融（Corporate Finance）：18%
- 交易与销售（Trading & Sales）：18%
- 零售银行业务（Retail Banking）：12%
- 商业银行业务（Commercial Banking）：15%
- 支付与清算（Payment & Settlement）：18%
- 代理服务（Agency Services）：15%
- 资产管理（Asset Management）：12%
- 零售经纪（Retail Brokerage）：12%

### 替代标准法（ASA）

在 SA 基础上，用 "贷款 × 因子" 替代零售银行业务的总收入：

```
零售银行业务资本 = 0.035 × 贷款敞口 + 0.025 × 存款余额
```

其他业务线仍用 SA 方法，能更好地反映银行实际业务规模。

### 高级计量法（AMA）

银行使用内部模型计量操作风险资本。

**要求：**
1. 使用内部数据和外部数据（行业数据池）
2. 使用情景分析（Scenario Analysis）
3. 覆盖所有 7 类操作风险事件
4. 通过监管验证

**方法：**
- 损失分布法（LDA）：拟合损失频率分布 + 损失强度分布
- 贝叶斯网络
- 极值理论（EVT）：专门建模尾部损失

```
资本 = VaR_{99.9%}（操作风险损失分布）
```

## 损失分布法（LDA）详解

### 核心逻辑

LDA 将操作风险损失分解为三步：

1. **损失频率分布（Loss Frequency）**：每年发生多少次操作风险事件？（泊松分布）
2. **损失强度分布（Loss Severity）**：每次事件的损失有多大？（对数正态分布、帕累托分布）
3. **蒙特卡洛模拟**：每年总损失 = Σ(各次事件损失)，重复 10000+ 次，取 99.9% 分位数作为资本要求。

### 分布选择

| 分布 | 适用场景 | 特点 |
|------|----------|------|
| **泊松分布** | 损失频率建模 | 稀有事件建模 |
| **对数正态分布** | 损失强度（中小损失） | 右偏、有上限 |
| **帕累托分布** | 损失强度（大损失/尾部） | 肥尾、适合极端事件 |
| **广义帕累托分布（GPD）** | 极值建模 | EVT 专用 |

### 数据来源

| 数据类型 | 说明 | 用途 |
|----------|------|------|
| **内部历史数据** | 本行过去 5-10 年操作风险事件记录 | 主要基础 |
| **外部数据** | 行业数据池（协和联盟 consortium） | 补充低频大损事件 |
| **情景分析** | 专家判断的极端场景 | 覆盖历史上未发生的风险 |
| **前瞻性数据** | 新业务、新产品、新流程风险评估 | 覆盖新兴风险 |

## 关键风险指标（KRI）

### 定义

关键风险指标（Key Risk Indicator）是通过可观测的运营指标来**提前预警**操作风险：

```
KRI = 可观测的运营指标 → 风险水平
```

**示例：**
- 系统可用率：99.5%（系统故障风险）
- 交易错误率：0.1%（执行风险）
- 员工离职率：15%（内部欺诈风险）
- 客户投诉率：0.5%（产品风险）

### 预警机制

| 信号 | 含义 | 行动 |
|------|------|------|
| 绿色 | 正常范围 | 持续监控 |
| 黄色 | 接近阈值 | 加强审查 |
| 红色 | 突破阈值 | 立即干预/升级 |

### 设计原则

- **可量化**：指标必须可以客观测量
- **前瞻性**：能在损失发生前发出预警
- **可操作性**：触发后有明确的应对流程
- **敏感性**：对真正的风险变化有响应

## 巴塞尔协议监管

### 监管演进

| 版本 | 修订内容 |
|------|----------|
| **Basel II（2004）** | 正式将操作风险纳入风险加权资产，引入 AMA |
| **Basel III（2010）** | 强化操作风险资本要求 |
| **Basel III.1 / FRTB（2019）** | 操作风险独立于市场风险框架，引入新的标准化计量方法 |

### Basel III.1 操作风险资本计量（新标准法）

Basel III.1（最终版）废除了 AMA，引入新的标准化方法：

```
操作风险资本 = min{ ILM × BIC, 72.5% × 上一年资本要求 }

其中：
BIC = Σ(各业务线收入 × β系数)（Business Indicator Component）
ILM = 1 - (Dividend Cont / BI)（Internal Loss Multiplier，红利分配比例调整）
```

**业务指标（Business Indicator）** = 利息净收入 + 非利息收入 + 金融工具净损益

巴塞尔协议II将操作风险纳入风险加权资产计算，要求银行计提操作风险资本。

## 实际案例

### 2012 摩根大通 London Whale

| 项目 | 内容 |
|------|------|
| 事件 | CIO 部门交易员 Bruno Iksil 大额CDS头寸亏损 |
| 损失金额 | ~60 亿美元 |
| 根本原因 | 模型风险（VaR 模型低估真实风险）+ 内部欺诈（隐瞒真实头寸） |
| 监管处罚 | 10 亿美元罚款 |
| 教训 | 操作风险不是孤立的——模型风险 + 欺诈 + 内控失效叠加 |

### 2015 瑞郎黑天鹅

| 项目 | 内容 |
|------|------|
| 事件 | 瑞士央行取消 EUR/CHF 汇率上限 |
| 损失 | Alpari UK、FXCM 等经纪商破产 |
| 损失类型 | 市场风险 + 操作风险（流动性管理失效） |
| 教训 | "不可能发生"的事件一旦发生，流动性风险急剧上升 |

## 操作风险与其他风险的交叉

| 交叉领域 | 说明 |
|----------|------|
| **模型风险** | 操作风险的重要子类别——错误模型导致错误决策 |
| **科技风险** | 网络攻击、系统故障属于操作风险 |
| **洗钱风险** | 外部欺诈 + 合规失败 |
| **声誉风险** | 操作风险事件引发的声誉损失（间接损失 > 直接损失） |
