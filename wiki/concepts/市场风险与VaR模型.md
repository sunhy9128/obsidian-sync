# 市场风险与VaR模型

> 市场风险计量、VaR体系与巴塞尔市场风险框架（FRTB）

**市场风险**（Market Risk）是指因利率、汇率、股票价格、商品价格等市场因子波动，导致金融资产价值损失的风险。**VaR**（Value at Risk，风险价值）是量化市场风险的核心指标。

---

## 一、VaR 的定义与表达

### 1.1 什么是 VaR

VaR 描述在一定置信水平下，在特定持有期内，投资组合的最大潜在损失：

```
VaR = 在 (1 - 置信水平) 的概率下，在持有期内可能遭受的最大损失

示例：
"1天99% VaR = 1000万"
→ 明天有99%的把握，损失不会超过1000万
→ 或者：明天有1%的概率，损失超过1000万
```

| 参数 | 常见取值 | 说明 |
|------|---------|------|
| **置信水平** | 95%、99%、99.5% | 越高越保守 |
| **持有期** | 1天、10天、1年 | 与流动性相关 |
| **分布假设** | 正态/历史/蒙特卡洛 | 影响计算精度 |

### 1.2 VaR 的局限性

| 缺陷 | 说明 | 应对 |
|------|------|------|
| **尾部风险盲区** | VaR 只关心阈值，不描述超阈值损失大小 | 引入 Expected Shortfall（ES） |
| **不满足次可加性** | 组合 VaR 可能大于各部分之和 | 使用 CoVaR、ES 替代 |
| **分布假设敏感性** | 正态假设低估肥尾 | 使用历史模拟或蒙特卡洛 |
| **稳定性问题** | VaR 对持有期不线性伸缩 | 需要换算因子 |

> **巴塞尔 IV 的改变**：巴塞尔 III 最终版（FRTB）将市场风险标准从 VaR 改为 **Expected Shortfall（ES）**，因为 ES 满足次可加性且能捕捉尾部风险。

## 二、VaR 计算方法

### 2.1 三种主流方法对比

| 方法 | 原理 | 优点 | 缺点 |
|------|------|------|------|
| **历史模拟法** | 用真实历史数据模拟未来分布 | 直观、无分布假设 | 依赖历史样本长度，忽视波动率聚集 |
| **参数法（方差-协方差）** | 假设正态分布，用均值/方差计算 | 计算快、易于分解 | 低估肥尾风险 |
| **蒙特卡洛模拟** | 随机生成大量场景计算收益分布 | 灵活、可处理非线性 | 计算量大、依赖模型假设 |

### 2.2 历史模拟法

```
步骤：
1. 收集过去 N 天的历史收益率数据（如 500 天）
2. 计算组合在每一天的盈亏（P&L）
3. 排序，找出第 1% 分位数（99% VaR）
4. 历史分布的分位数即为 VaR

改进：GARCH 加权历史模拟
- 近期数据赋予更高权重
- 反映波动率聚集效应
```

### 2.3 参数法（正态分布假设）

```
VaR(99%, 1天) = z_{0.99} × σ × 组合价值

其中 z_{0.99} = 2.33（标准正态分位数）

示例：
组合价值 = 1亿，σ = 1%（日波动率）
VaR = 2.33 × 1% × 1亿 = 233万

注：1天VaR 转 10天VaR：通常 × √10 ≈ 3.16
```

### 2.4 蒙特卡洛模拟

```
步骤：
1. 设定随机过程（如几何布朗运动）：
   dS = μSdt + σSdW
2. 随机生成 N 条资产价格路径（10000+ 次）
3. 计算每条路径的组合价值
4. 汇总分布，计算 VaR

优势：可处理期权等非线性产品、复杂相关结构
劣势：计算耗时，模型风险大
```

## 三、Expected Shortfall（ES）

### 3.1 定义

ES（期望尾部损失）是 VaR 的改进指标，定义为**超过 VaR 阈值的平均损失**：

```
ES_{99%} = E[Loss | Loss > VaR_{99%}]

含义：超过 VaR 的那些极端情况下，平均损失是多少
```

| 指标 | 描述 | 特点 |
|------|------|------|
| **VaR（99%）** | 1% 的概率损失超过 X | 只报告阈值，不报尾部大小 |
| **ES（99%）** | 超阈值的平均损失为 Y | 捕捉尾部严重程度 |

> ES 比 VaR 更保守——2008 年金融危机中，使用 VaR 的银行严重低估了实际损失。

### 3.2 ES 的数学表达

```
ES_α = (1/(1-α)) × ∫_α^1 VaR_u du

即 VaR 在 [α, 1] 区间上的平均值的负值
```

## 四、巴塞尔市场风险框架（FRTB）

### 4.1 历史演变

| 框架 | 时间 | 核心方法 |
|------|------|---------|
| 1996 年修订 | 1996 | 旧版 VaR（10天99%） |
| 巴塞尔 II.5 | 2009 | 压力 VaR（Stress VaR）叠加 |
| FRTB（Basel III.1） | 2019（执行中） | **Expected Shortfall + 流动性调整** |

### 4.2 FRTB 的核心变化

| 变化 | 旧版（Basel II） | FRTB（Basel III.1） |
|------|-----------------|---------------------|
| **核心指标** | 10天 99% VaR | 10天 97.5% Expected Shortfall |
| **资本下限** | 旧版 VaR × 3 + Stress VaR | ES × 乘数因子（≥3）|
| **风险因子细分** | 7 个风险类别 | **10 个风险因子 + 114 个风险子类** |
| **内模法限制** | 可对冲 | 交易台层面严格限制 |
| **流动性调整** | 无 | 明确持有期（1天/10天/20天/60天）|

### 4.3 FRTB 风险因子体系

```
FRTB 将风险细分为 10 个风险类别（Risk Class）：
1. GIRR（一般利率风险）— 19 个货币分区
2. FX（外汇风险）— 10 个货币对
3. Equity（股票风险）— 27 个市场/指数
4. Commodity（大宗商品）— 11 个商品
5. Credit Spread（非主权信用价差）— Sovereign / NSCCD 分类
6. Securitization（资产证券化）
7. Option（期权风险）— 非线性

每个风险类别内进一步细分子类（如 GIRR 按货币、按曲线点）
```

### 4.4 内模法（IMA）vs 标准法（SA）

| 维度 | 内模法（IMA） | 标准法（SA） |
|------|-------------|-------------|
| **适用对象** | 满足条件的交易台 | 所有机构 |
| **资本要求** | ES × 乘数因子（mC） | 敏感性 × 风险权重 |
| **对冲认可** | 高度认可（同风险因子内对冲） | 有限认可 |
| **门槛要求** | 交易台层面需达到 80% 交易笔数可建模 | 无门槛 |
| **限制条件** | 需每日计算 ES，不得超过 SA 的 1.5 倍 | — |

```
乘数因子 mC = 3 + add-on
  add-on = max(0, 10% × Backtesting exception count - 2)

Backtesting 异常次数越多，乘数因子越高
```

## 五、Backtesting（回溯测试）

### 5.1 概念

Backtesting 将 VaR/ES 模型预测与实际损益对比，检验模型的准确性。

```
例外事件：实际损失 > VaR 预测值

绿色区域（99% 置信）：0-4 次例外 → 模型正常
黄色区域：5-9 次例外 → 需关注，监管约谈
红色区域：10+ 次例外 → 模型失效，资本惩罚
```

### 5.2 巴塞尔对 Backtesting 的处置

| 例外次数（250交易日） | 区域 | 处置 |
|----------------------|------|------|
| 0-4 | 绿色 | 正常 |
| 5 | 黄色 | 监管约谈 |
| 6-9 | 黄色 | 增加乘数因子 |
| ≥10 | 红色 | 模型不可用，资本惩罚 |

### 5.3 注意事项

- Backtesting 基于**日损益**，不是日收益率
- 实践中应区分**模型失效**（系统偏差）与**市场异常**（黑天鹅）
- 巴塞尔 III.1 将 Backtesting 用于 **ES 模型的乘数因子校准**

## 六、Stressed VaR（压力 VaR）

### 6.1 概念

压力 VaR 是用压力期（如 2008 金融危机、2010 欧债危机）数据计算的 VaR，作为资本附加项：

```
市场风险资本 = ES（正常期）× 乘数因子 + Stress VaR

→ 确保银行在市场极端时期也有足够资本
```

### 6.2 压力期选择

| 危机事件 | 时间 | 对应的风险因子 |
|----------|------|---------------|
| 全球金融危机 | 2008.9-12 | 信用利差、股票、流动性 |
| 欧债危机 | 2011 | 主权利差、外汇 |
| 新兴市场危机 | 1997-98 | 外汇、股票 |
| 中国股灾 | 2015 | A股、人民币汇率 |

> 巴塞尔要求银行定期审查压力期，确保能反映当前系统性风险。

## 七、风险因子与风险分解

### 7.1 主要市场风险因子

| 风险类别 | 主要因子 | 典型敏感度指标 |
|----------|----------|--------------|
| **利率风险** | 国债收益率曲线平行移动、陡峭化/平坦化 | DV01（Dollar Value of 1bp） |
| **外汇风险** | 即期汇率变动 | FX 敞口（各货币净头寸） |
| **股票风险** | 股指价格、个股波动率 | Delta、Gamma |
| **商品风险** | 原油、黄金、铜等 | 久期、展期成本 |
| **信用利差风险** | 信用利差扩大/收窄 | CS01（Credit Spread × Duration） |

### 7.2 组合敏感度分解

```
总 VaR = Σ_i β_i × VaR_i + Σ_{i≠j} ρ_{ij} × Cov(i,j)

分解维度：
- 按风险类别：利率VaR + 外汇VaR + 股票VaR + 商品VaR
- 按业务条线：交易台A + 交易台B + 自营账户
- 按地区：亚太 + 欧美 + 新兴市场
```

## 八、实践中的常见问题

### 8.1 杠杆效应与非线性风险

- **期权、结构的非线性产品**：不能用简单的 delta VaR，需用全定价或 delta-gamma 近似
- **路径依赖产品**：障碍期权、亚式期权需特殊处理

### 8.2 流动性调整

FRTB 引入流动性调整持有期（LUHP），根据资产流动性确定：

| 流动性等级 | 资产类型 | 持有期 |
|------------|----------|--------|
| 高流动性 | 主权利率、货币市场 | 10 天 |
| 中流动性 | 投资级信用债、股指 | 20 天 |
| 低流动性 | 新兴市场债券、结构化产品 | 60 天 |

### 8.3 相关性风险

- **市场相关性在压力下趋于 1**（"All correlations go to 1 in a crisis"）
- FRTB 引入**相关性应力测试**（Correlation Stress）：检验组合在相关性系统性上升时的表现
