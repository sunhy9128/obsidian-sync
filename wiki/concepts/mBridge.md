# mBridge（多边央行数字货币桥）

> Multi-CBDC Bridge — 重构全球跨境支付格局的"数字货币联盟"

**mBridge**（Project mBridge，多边央行数字货币桥项目）是由国际清算银行（BIS）创新中心、香港金融管理局、泰国央行、阿联酋中央银行和中国人民银行数字货币研究所共同发起的多边央行数字货币（CBDC）跨境支付实验平台。它旨在**建立一个基于分布式账本技术（DLT）的多边 CBDC 清算平台，实现近乎即时、低成本、点对点的跨境支付**，并可能在未来彻底改变以 SWIFT + CHIPS 为核心的现有跨境支付格局。

mBridge 是数字货币时代"非美元清算"的最大试验，也是 CIPS 数字化升级的方向。

## 二、核心定义与本质

### 2.1 定义

mBridge 是**由多个国家的央行共同运营的多边 CBDC 跨境支付平台**，它通过 DLT 技术实现：

```
【mBridge 的核心机制】
├── 1. 每个参与方发行自己的 CBDC
├── 2. CBDC 在 mBridge 平台上"代币化"
├── 3. 跨境支付通过"桥接"完成
├── 4. 各央行直接清算（不需要 SWIFT + CHIPS）
└── 5. 实现近乎即时、低成本、点对点的跨境支付
```

### 2.2 mBridge 的本质

```
【mBridge 的本质】

传统跨境支付：中介化、多层级、慢速、昂贵
├── sender → 代理行 A → 代理行 B → receiver
├── 时间：3-5 个工作日
├── 成本：50-300 美元 + 汇率损失
└── 摩擦点：每个代理行都是潜在故障点

mBridge 跨境支付：去中介化、点对点、快速、低成本
├── sender CBDC → mBridge → receiver CBDC
├── 时间：秒级（理论上接近即时）
├── 成本：接近零
└── 摩擦点：央行直接清算，无中介
```

### 2.3 mBridge vs CIPS 的关系

| 维度 | mBridge | CIPS |
|------|---------|------|
| 清算对象 | CBDC（数字化央行货币）| 传统账户（人民币存款）|
| 技术基础 | 分布式账本技术（DLT）| 中心化账户系统 |
| 参与方 | 多边央行联盟 | 中国主导 + 国际参与 |
| 货币 | 多币种 | 人民币为主 |
| 跨境模式 | 点对点（去中介）| 多级代理行（有中介）|
| 时间 | 秒级 | 分钟到小时 |
| 与 SWIFT 关系 | 替代（理论上）| 协同（多数情况下仍需 SWIFT 报文）|
| 战略意义 | 跨境支付的"下一代表明 | 现有清算系统的"基础设施升级" |

## 四、mBridge 的技术架构

### 4.1 三层架构

```
【mBridge 技术架构】

Layer 1：CBDC 发行层
├── 各参与方央行发行自己的 CBDC
├── 中国 e-CNY、香港 e-HKD、泰国 CBDC 等
└── CBDC 在自家体系内运行

Layer 2：mBridge 桥接层
├── 基于 DLT（Hashgraph）
├── 各 CBDC 在桥接层"代币化"
├── 跨境支付通过桥接完成
└── 各央行节点验证交易

Layer 3：应用层
├── 国际贸易结算
├── 跨境汇款
├── 资本市场交易
└── 其他跨境支付场景
```

### 4.2 关键技术特征

```
【特征 ①：DLT（分布式账本技术）】
├── 所有参与央行节点共同维护账本
├── 交易验证需要多数节点同意
└── 无需中心化清算机构

【特征 ②：央行直接清算】
├── 各央行在 mBridge 上开设账户
├── 跨境支付直接清算
└── 不需要 SWIFT 报文 + 代理行清算

【特征 ③：货币桥接】
├── 一国 CBDC 在 mBridge 上"映射"
├── 跨境支付通过桥接完成
└── 不需要经过传统外汇市场

【特征 ④：监管嵌入】
├── 合规检查嵌入协议层（KYC/AML）
├── 反洗钱实时监控
└── 央行实时监管
```

### 4.3 跨境支付完整流程

```
【mBridge 跨境支付流程（示例：中泰贸易）】

1. 中国进口商从中国央行获取 e-CNY 代币
2. 在 mBridge 平台上发起支付请求
3. mBridge 平台自动验证（中国央行 + 泰国央行节点）
4. e-CNY 转换为 e-THB（桥接）
5. 泰国出口商收到 e-THB 代币
6. 泰国出口商可兑换为泰国本地银行存款

总耗时：秒级
总成本：接近零（只需支付平台使用费）
中间环节：无（点对点）
```

## 五、mBridge 的战略意义

### 5.1 对全球金融体系的颠覆

```
【颠覆 ①：去 SWIFT 化】
├── SWIFT 是报文系统（信息流）
├── mBridge 是支付 + 清算（资金流）
└── mBridge 理论上可以完全绕开 SWIFT

【颠覆 ②：去 CHIPS 化】
├── CHIPS 是美元清算（资金流）
├── mBridge 可以直接清算 CBDC
└── 美元清算的"中介"地位被削弱

【颠覆 ③：去代理行化】
├── 传统：sender → 代理行 A → 代理行 B → receiver
├── mBridge：sender → mBridge → receiver
└── 代理行（多在纽约/伦敦）的作用被削弱

【颠覆 ④：去美元中介化】
├── 美元是全球跨境支付的"中介货币"
├── mBridge 实现 CBDC 直接清算
└── 美元中介角色被技术削弱
```

### 5.2 对各国央行的吸引力

```
【对发达经济体】
├── 提升支付效率
├── 降低跨境支付成本
└── 维持货币国际地位

【对新兴市场】
├── 减少对美元依赖
├── 提升本币国际化
├── 应对美国长臂管辖
└── 推动区域一体化

【对中小国家】
├── 接入多边清算系统
├── 避免被边缘化
└── 降低对大国的依赖
```

### 5.3 对中国的特殊价值

```
【mBridge 对中国的战略意义】

1. **e-CNY 的国际化通道**
   └── mBridge 让 e-CNY 跨境使用有了制度框架

2. **CIPS 的数字化升级**
   └── mBridge 是 CIPS 的"下一代版本"

3. **反制长臂管辖**
   └── mBridge 难以被美国长臂管辖（数字货币跨境难追踪）

4. **人民币国际化的"弯道超车"**
   └── 数字货币可能重塑国际货币格局

5. **一带一路的金融基础设施**
   └── 沿线国家可以通过 mBridge 进行本币结算

6. **与美元体系的"非对称竞争"**
   └── 美国尚未推出数字美元，中国可抢占先机
```

## 六、mBridge 与其他清算系统的对比

### 6.1 全球清算基础设施矩阵

| 系统 | 主导方 | 清算对象 | 技术 | 与美元关系 |
|------|--------|---------|------|-----------|
| **SWIFT + CHIPS** | 美欧 | 美元（CHIPS）| 报文 + 中心化清算 | 美元主导 |
| **CIPS** | 中国 | 人民币 | 报文 + 中心化清算 | 替代美元清算 |
| **SPFS** | 俄罗斯 | 卢布 | 报文 + 中心化清算 | 替代美元清算 |
| **INSTEX** | 欧盟 | 欧元（易货）| SPV 易货 | 绕开美元 |
| **mBridge** | 多边央行 | 多币种 CBDC | DLT 分布式账本 | 不绑定单一货币 |

### 6.2 mBridge 的"非对称优势"

```
【mBridge vs 现有清算系统】

传统清算系统的局限：
├── 多级代理行 → 慢、贵、易出错
├── 依赖 SWIFT 报文 → 受美国长臂管辖影响
├── 美元清算主导 → 美元依赖
└── 不透明 → 反洗钱成本高

mBridge 的优势：
├── 点对点 → 快、便宜、低摩擦
├── 不依赖 SWIFT → 规避长臂管辖
├── 多币种清算 → 不绑定美元
└── DLT 透明 → 反洗钱实时监控
```

## 八、mBridge 的挑战与风险

### 8.1 五大挑战

```
【挑战 ①：技术成熟度】
├── DLT 大规模应用仍在验证
├── 可扩展性（TPS）需进一步提升
├── 与各国央行系统的对接复杂
└── 网络安全风险（数字货币特性）

【挑战 ②：法律与监管】
├── CBDC 的法律地位各国不一
├── 跨境支付的监管协调困难
├── 反洗钱标准统一需要时间
└── 法律责任划分复杂

【挑战 ③：参与方的扩展性】
├── 创始 4 方 → 扩展需要更多国家加入
├── 政治阻力（美国对盟友施压）
├── 标准制定需要共识
└── 商业可持续性待验证

【挑战 ④：与现有系统的兼容性】
├── 与 SWIFT / CIPS / SPFS 的关系
├── 与传统银行账户体系的衔接
├── 过渡期的"双轨"运行成本
└── 何时完全替代旧系统

【挑战 ⑤：地缘政治博弈】
├── 美国的反对（数字美元尚未推出）
├── 欧盟的观望（数字欧元研发中）
├── 部分国家的"选边站"压力
└── mBridge 可能被视为"中国主导"
```

### 8.2 美国的担忧与反应

```
【美国对 mBridge 的担忧】

1. **削弱美元霸权**
   └── CBDC 直接清算削弱美元中介地位

2. **绕过 SWIFT**
   └── mBridge 可以完全绕开 SWIFT

3. **难以长臂管辖**
   └── 数字货币跨境难追踪，长臂管辖难以延伸

4. **失去监管权**
   └── 央行直接清算，美国难以执行制裁

5. **数字美元落后**
   └── 数字美元仍在讨论阶段，远落后于 mBridge

【美国可能的应对】
├── 加速数字美元研发
├── 对盟友施压不加入 mBridge
├── 推动 SWIFT 数字化升级
├── 利用监管优势限制数字美元使用
└── 寻求技术上的弯道超车
```

## 十、关键洞察

> [!key-insight] mBridge 是"数字货币时代的非美元清算"
>
> CIPS 是"传统清算基础设施的非美元版本"，mBridge 是"数字货币清算基础设施"。两者代表了两个时代，CIPS 是过渡，mBridge 是未来。

> [!key-insight] mBridge 的"非对称反制"价值
>
> 美国的长臂管辖难以延伸到数字货币领域：传统美元清算可以追踪，但 CBDC 跨境支付的去中心化特性让追踪难度大幅提升。这是 mBridge 对中国的战略价值。

> [!contradiction] 数字货币的"悖论"
>
> 数字货币让跨境支付更高效、更便宜，但同时也让监管更难（反洗钱、KYC）、让长臂管辖更难执行。这种"双刃剑"特性决定了 mBridge 既是机遇也是挑战。

> [!key-insight] mBridge 的"去中心化"不等于"无政府"
>
> mBridge 由各国央行共同运营，每个央行节点都有完全的监管权。这是一种"多中心化"而非"无中心化"——既保留了去中心化的效率，又维持了中心化的监管。

```
【mBridge + CIPS 双轨格局】

传统账户清算（CIPS）：
├── 适用于：现有银行体系、传统外贸
├── 优势：成熟、稳定、广泛使用
└── 局限：慢、贵、依赖 SWIFT

数字货币清算（mBridge）：
├── 适用于：CBDC 持有者、新型跨境支付
├── 优势：快、便宜、去中介化
└── 局限：仍在发展、参与方有限

【双轨关系】
├── 短期：mBridge 是 CIPS 的补充
├── 中期：mBridge 与 CIPS 形成竞争
├── 长期：mBridge 可能主导，SWIFT 边缘化
└── 关键变量：参与方扩展速度 + 数字货币普及度
```
