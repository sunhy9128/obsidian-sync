# 央行数字货币

## 概述

央行数字货币（Central Bank Digital Currency / CBDC）是由中央银行发行和背书的数字化法定货币。与比特币等私人加密货币不同，CBDC 是中央银行的直接负债（等同于现金和准备金），具有法定货币地位。CBDC 被视为 21 世纪国际货币体系的潜在"游戏规则改变者"——可能重塑跨境支付、货币政策和国际货币竞争格局。

---

## 一、CBDC 在货币谱系中的位置

```
                    ↓ 可用性 ↑
                    
     CBDC（零售）    ←  面向公众
          ↑
     CBDC（批发）    ←  面向金融机构
          ↑
      准备金        ←  银行在央行存款
          ↑
      现金/硬币      ←  实物货币

         ↑ 技术化程度 ↑
```

| 货币形态 | 发行主体 | 形式 | 是否计息 | 匿名性 | 技术基础 |
|----------|----------|------|----------|--------|----------|
| 现金 | 央行 | 物理 | 否 | 高 | 无 |
| 准备金 | 央行 | 电子（仅限银行） | 是 | 极低 | 央行系统 |
| **零售 CBDC** | **央行** | **电子（公众）** | **可设计** | **可设计** | **DLT/传统** |
| 银行存款 | 商业银行 | 电子 | 是 | 低 | 银行系统 |
| 稳定币 | 私人机构 | 电子 | 否 | 可设计 | DLT |
| 加密货币 | 无 | 电子 | 否 | 伪匿名 | DLT |

## 二、CBDC 的驱动力与设计选择

### 各国发行 CBDC 的动机

| 动机 | 发达经济体 | 新兴市场 | 中国 |
|------|-----------|---------|------|
| **支付效率** | 降低现金使用成本 | 提升金融普惠 | 提升支付效率 |
| **货币政策** | 突破"零利率下限" | 改善传导机制 | 增强传导精度 |
| **反洗钱/反恐融资** | 提高交易可见性 | 加强监管 | 加强监管 |
| **去美元化** | 不适用 | 减少美元依赖 | 核心目标之一 |
| **国际化** | 维持货币地位 | 提升国际影响力 | **人民币国际化的"弯道超车"** |
| **现金替代** | 现金使用下降 | 替代高成本现金 | 现金已基本被电子支付替代 |

### 关键设计维度

```
CBDC 设计选择树
├── 1. 接入范围
│   ├── 零售（公众可用）—— 中国 e-CNY、瑞典 e-Krona
│   └── 批发（仅机构）—— 新加坡 Ubin、加拿大 Jasper
├── 2. 架构
│   ├── 单层（央行直接与公众交易）—— 风险大，很少采用
│   └── 双层（央行→商业银行→公众）—— 主流选择
├── 3. 计息
│   ├── 计息（类似准备金利率）—— 影响货币政策传导
│   └── 不计息（类似现金）—— 减少对银行体系的冲击
├── 4. 额度限制
│   ├── 有（如 e-CNY 设个人限额）—— 防止银行存款流失
│   └── 无—— 可能引发"数字挤兑"
├── 5. 匿名性
│   ├── 完全匿名（数字现金）
│   ├── 分级匿名（小额匿名，大额可追踪）—— e-CNY 方案
│   └── 完全可追踪（所有交易可查）
└── 6. 跨境兼容
    ├── 独立运行（各国 CBDC 互不连接）
    ├── 桥接（CBDC 通过共同平台互通）
    └── 多边共享（mBridge 模式）
```

## 四、数字人民币 e-CNY

### 定位

e-CNY（数字人民币）是全球最大、最领先的零售 CBDC 项目，由中国人民银行推出。它**不是**加密资产，而是**数字化的人民币现金**（M0）。

### 核心参数

| 维度 | e-CNY |
|------|-------|
| 发行方 | 中国人民银行 |
| 定位 | M0（流通中现金），不计息 |
| 架构 | 双层运营（央行→商业银行→公众） |
| 上线 | 2019 年底试点，2023 年扩大至 26+ 城市 |
| 累计交易额（2025） | ~1.5 万亿元人民币 |
| 个人限额 | 按 KYC 等级（最低 1 万，最高 50 万） |
| 匿名性 | "可控匿名"——小额匿名，大额可追踪 |
| 技术 | 混合架构（部分中心化 + 部分分布式） |

### e-CNY vs 现有电子支付

| 对比 | e-CNY | 微信支付/支付宝 |
|------|-------|----------------|
| 性质 | 央行货币（M0） | 商业银行货币（M1/M2） |
| 账户依赖 | 不需要银行账户（"松耦合"） | 需要绑定银行卡 |
| 离线支付 | 支持（NFC 碰一碰） | 有限 |
| 提现费用 | 零（等同现金） | 可能有手续费 |
| 可编程性 | 可附加智能合约（条件支付） | 有限 |
| 跨境 | 设计为易于跨境 | 依赖境内账户 |

### e-CNY 的跨境战略

e-CNY 对人民币国际化有特殊意义：

```
国内层面：替代现金、提升支付效率、增强监管能力
  ↓
区域层面：mBridge（多边桥接）+ 东盟本币结算
  ↓
全球层面：人民币国际化的"数字通道"
```

**mBridge（多边央行数字货币桥项目）**：
- 参与方：中国香港金管局、泰国央行、阿联酋央行、中国人民银行数字货币研究所
- 目标：建立多边 CBDC 跨境支付平台，实现**近乎即时、低成本、点对点**的跨境支付
- 2024 年进入 MVP（最小可行产品）阶段
- 潜在影响：削弱对 SWIFT + CHIPS 美元的依赖

> [!contradiction] e-CNY 的悖论
> - **国内成效显著**：试点不断扩大，技术成熟
> - **国际推广有限**：各国有政治顾虑（数据安全、监控）
> - **跨境效率提升**：但需要对应的制度安排（货币互换协议、清算行）
> - **对 SWIFT 的替代有限**：SWIFT 是报文系统，e-CNY 是支付结算系统——两者不完全竞争

## 五、CBDC 对国际金融体系的影响

### 对跨境支付

```
当前跨境支付（慢、贵、不透明）
  sender → 代理行 A → 代理行 B → receiver
  ├── 时间：3-5 个工作日
  ├── 成本：5-10%（小额汇款达 10%）
  └── 追踪：困难

CBDC 跨境支付（快、便宜、透明）
  sender CBDC → mBridge → receiver CBDC
  ├── 时间：几乎实时
  ├── 成本：接近零
  └── 追踪：实时可见
```

### 对美元霸权

| 影响方向 | 短期（1-5年） | 中期（5-10年） | 长期（10年+） |
|----------|-------------|--------------|-------------|
| 美元储备货币地位 | 无影响 | 边际影响 | 取决于技术和制度的成熟度 |
| SWIFT 替代 | 不可能 | 局部替代（中俄、一带一路沿线） | 可能在区域内形成替代网络 |
| 美元定价 | 无影响 | 无影响 | 无影响（CBDC 不改变计价单位） |
| 跨境结算 | 补充 | 竞争 | 多元并存 |

> [!key-insight] CBDC 不是"美元杀手"
> CBDC 改变的是**支付基础设施**（通道），而非**计价单位和价值储藏**（储备货币）。即使跨境支付变得更快更便宜，只要国际贸易仍然以美元定价、各国央行仍然信任美元储备价值，美元霸权就不会被 CBDC 直接动摇。

### 对货币政策

| 能力 | 无 CBDC | 有 CBDC |
|------|---------|---------|
| 负利率传导 | 困难（人们可以持有现金） | 容易（可以取消现金或对 CBDC 收息） |
| 定向刺激 | 困难（只能全面放水） | 容易（"直升机撒钱"的可编程版本） |
| 实时监控 | 滞后 | 实时（但隐私问题） |
| 金融稳定 | 数字挤兑是想象 | "数字挤兑"容易但额度限制可设计 |

## 六、挑战与风险

| 风险 | 说明 | 应对措施 |
|------|------|----------|
| **银行存款流失** | 公众将存款转为 CBDC，银行失去稳定融资 | 额度限制 + 不计息 |
| **数字挤兑** | 危机时一键将全部存款转为 CBDC | 额度限制 + 银行流动性支持 |
| **隐私问题** | 央行可查看所有交易记录 | 可控匿名 + 分级隐私 |
| **跨境兼容性** | 各国 CBDC 标准不一，互不兼容 | 国际标准（BIS）+ mBridge |
| **网络安全** | 单点故障风险（央行系统被攻击） | 多层安全 + 离线方案 |
| **银行脱媒** | 央行直接与公众竞争存款 | 双层架构（不直接面向公众零售） |
